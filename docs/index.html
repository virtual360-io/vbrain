vbrain

The knowledge base that knows
what you know — and who you are

A personal knowledge base reduced to its essence: Claude Code skills, a single deterministic Go binary, and SQLite FTS5. Markdown is the source of truth; the model only steps in for judgment.

No Ruby, no gems, no runtime to install — one self-contained binary for Linux, macOS, and Windows.

The premise

A wiki you own, an index you can throw away, and a model kept on a short leash.

📄

Markdown is the truth

Your knowledge lives as plain markdown with YAML frontmatter, connected by [[wikilinks]]. Readable in any editor, on GitHub, in Obsidian.

🗃️

SQLite is disposable

The FTS5 index is derived: delete db/ and vbrain reindex rebuilds everything from the wiki — graph included.

⚙️

Code does the deterministic work

Detecting sources, normalizing queries, writing frontmatter, building SQL — all tested Go. If code can answer, code answers.

🧠

The model only judges

Chunking, synthesizing pages, consolidating identity — the few tasks that genuinely need judgment. Nothing else.

★ the soul layer

Knowing is not believing

You can read Friedman and Marx without believing either — and you don't believe the same thing your whole life. So vbrain keeps a second layer: wiki/_soul/, a small set of pages describing how and why you act — the layer an agent consults before deciding anything in your name.

A daily routine looks at your recent actions — what you asked, decided, wrote — cross-references them with what you know, and consolidates your identity. Ranking is intent-aware: acting outranks knowing. Soul pages are favored by default, and for decision or belief questions they have absolute precedence over any mere fact.

Lean. A core memory earns its place by recurring across actions — not by being mentioned once.
No silent contradiction. A belief cycle is either resolved with explicit context, or one belief is stale and pruned.
Beliefs are mortal. What you abandoned is deleted or rewritten as "formerly X, now Y, because…".
Grounded. Every soul page traces to a real signal. If it can't be grounded, it isn't written.

How a note gets in

A file or URL flows through one deterministic pipeline; the wiki is never written by hand.

ingest → raw/→ chunk (LLM)→ write pages (LLM)→ write-pages→ reindex→ commit

Knowledge pages are written only by vbrain write-pages; the soul layer only by vbrain soul-write, after the routine consolidates. Skills never touch markdown directly.

The skills

The Claude Code interface — slash commands that call the vbrain binary.

CommandWhat it does
/vbrain-add-knowledgeIngest a file/URL → chunk → generate grounded pages → reindex → commit.
/vbrain-query-knowledgeAsk in natural language; answers from your notes without exposing internals. Decision/belief questions defer to the soul.
/vbrain-add-realtime-knowledgeConnect a live source — Google Calendar, Gmail, Slack — via MCP.
/vbrain-add-routineAdd a cron-scheduled routine with its own prompt.
/vbrain-routineWatch loop: fire due routines (like the nightly dream and daily soul).

Install

Download the binary for your platform, then let it bootstrap everything.

# e.g. Linux x86-64 — pick your asset from the latest release
curl -L -o vbrain https://github.com/virtual360-io/vbrain/releases/latest/download/vbrain-linux-amd64
chmod +x vbrain
./vbrain install        # PATH + skills + base bootstrap + GitHub onboarding

Assets: vbrain-linux-amd64, vbrain-linux-arm64, vbrain-darwin-amd64, vbrain-darwin-arm64, vbrain-windows-amd64.exe. Update anytime with vbrain update.

# try it
printf "# Postgres\n\nUse REPLICA IDENTITY FULL for logical replication.\n" > /tmp/pg.md
vbrain ingest /tmp/pg.md
vbrain reindex
vbrain query "replica identity" --format markdown   # → your note, ranked
